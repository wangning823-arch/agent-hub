# 更新日志

## 2026-06-13

### 安全加固

- **WebSocket 认证修复**: 无 `.token` 文件时允许无认证连接，避免合法用户被拒
- **CORS 限制收紧**: 限制允许的来源域名，防止跨域请求滥用
- **安全响应头**: 添加 X-Content-Type-Options、X-Frame-Options、X-XSS-Protection 等 HTTP 安全头
- **XSS 防护**: 对用户输入和消息内容进行转义处理
- **登录限流**: 添加登录接口速率限制，防止暴力破解
- **路径验证加固**: 增强文件操作的路径校验，防止目录遍历攻击

### 性能优化

- **数据库防抖保存**: 使用防抖机制合并高频写入，减少 SQLite 磁盘 I/O
- **批量 API 查询**: 优化会话列表和消息加载的数据库查询，减少 N+1 问题
- **React.memo 优化**: 对高频渲染组件添加 memo 包裹，减少不必要的重渲染
- **文件列表懒加载**: 大目录文件列表按需加载，避免一次性加载过多文件

### 目标监控改进

- **完成关键词检测优化**: 改进任务完成判断逻辑，支持更多完成标志词
- **isWorking 状态修复**: 修复 agent 停止后 `isWorking` 未重置导致后续消息被拒绝的问题
- **isRunning 状态重置**: 修复 agent 异常退出后状态未清理的问题
- **LLM 完成判断增强**: 明确完成标准 — 所有修改 + 所有测试 + 功能验证，仅生成计划文件不算完成
- **超时保护**: 添加 5 分钟超时检测，防止监控无限等待

### Mimo Server 模式

- **多会话并发支持**: 新增 `mimo serve` 常驻进程管理，所有 mimo agent 通过 `--attach` 共享同一 server
- **完整上下文恢复**: Server 模式下 `--session` 安全可用，重启后恢复完整对话历史（包括工具调用）
- **多项目支持**: 单个 server 通过 `--dir` 支持不同项目目录的会话
- **自动生命周期管理**: Server 自动启动、健康检查（30s）、崩溃自动重启
- **优雅降级**: Server 启动失败时自动回退到直接模式（`--pure`），不影响使用
- **进程退出清理**: agent-hub 关闭时自动关闭 mimo server 进程

### 会话持久化修复

- **mimo 会话恢复**: 修复服务器重启后 mimo 丢失上下文的问题
- **pendingHistory 改进**: mimo/opencode 重启后始终注入历史上下文（最后 10 条消息 + 摘要）
- **--session 挂起修复**: 避免 TUI 进程持有 SQLite 锁时 `--session` 导致的无限挂起

### Bug 修复

- 修复文件上传中文文件名乱码问题
- 修复工作流引擎已完成步骤可能被重复执行的问题
- 修复旧版工作流依赖字段使用名称而非 ID 导致崩溃的问题
- 修复 `/context` 命令超时问题（15s → 60s）
